# Configuration

## Environment Variables

### Genius API Token

```bash
export GENIUS_ACCESS_TOKEN="your_token_here"
```

This enables Genius as a fallback lyrics source when LRCLib doesn't have synced lyrics.

### Whisper Model Cache

By default, Whisper models are downloaded to `~/.cache/huggingface/hub/`. You can change this:

```bash
export HF_HOME="/path/to/cache"
```

## Command Line Options

All command line options can be set as defaults by creating a configuration file.

### pyproject.toml

Add to your `pyproject.toml`:

```toml
[tool.lrcgen]
model-size = "small"
device = "cpu"
recursive = true
```

## Whisper Models

### Model Comparison

| Model | Size | Speed | Accuracy | Use Case |
|-------|------|-------|----------|----------|
| tiny | ~39MB | Fastest | Basic | Quick drafts |
| base | ~74MB | Fast | Good | General use |
| small | ~244MB | Medium | Better | Recommended default |
| medium | ~769MB | Slow | Great | High accuracy |
| large-v3 | ~1.5GB | Slowest | Best | Maximum quality |

### GPU vs CPU

**CPU Processing:**
```bash
lrcgen music/ --device cpu
```

**GPU Processing (requires CUDA):**
```bash
lrcgen music/ --device cuda
```

GPU is significantly faster for large batches.

## Lyrics Sources

### LRCLib (Primary)

- Free, no API key required
- Provides synced lyrics (with timestamps)
- Community-maintained database

### Genius (Fallback)

- Requires API token
- Provides plain text lyrics only
- Timestamps generated by Whisper

### Processing Priority

1. LRCLib with synced lyrics → Use directly
2. LRCLib with plain lyrics → Use Whisper for timestamps
3. Genius lyrics → Use Whisper for timestamps
4. No lyrics found → Use Whisper transcription only

## Performance Tuning

### Batch Processing

For large collections, consider:

1. Use a smaller model for initial scan
2. Use GPU acceleration
3. Process overnight for large libraries

### Memory Usage

- `tiny` model: ~100MB RAM
- `base` model: ~150MB RAM
- `small` model: ~300MB RAM
- `medium` model: ~800MB RAM
- `large-v3` model: ~2GB RAM
